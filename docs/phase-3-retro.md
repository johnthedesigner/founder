## What was harder than expected

**Playwright test isolation for authenticated project creation.** `page.request.post(BACKEND_URL + '/projects', ...)` and `page.request.post(FRONTEND_URL + '/projects', ...)` both failed silently â€” the projects were created anonymously (no `userId`) even after browser login, because `page.request` doesn't reliably share the session cookie from the browser context across origins. The fix â€” `page.evaluate(async () => fetch('/projects', { credentials: 'include' }))` â€” runs the fetch inside the browser's JS context at the app origin, which guarantees the Vite proxy receives the cookie correctly.

**`userStore.isLoading` starting as `false` was a latent bug.** Home page immediately redirected to `/login` on any full page reload (`page.goto('/')`) because the redirect effect fired with initial state `{user: null, isLoading: false}` before `fetchUser()` could set `isLoading: true`. This had never surfaced in earlier tests because tests always navigated via React Router (store intact) rather than full reloads. The fix (initialize `isLoading: true`) is correct and all tests continued to pass.

**Dev servers on test ports causing false failures.** Already documented from Phase 2. The standing rule (kill ports 3001/5299/5180 before any Playwright run) was applied correctly throughout this phase.

## Key decisions

| Decision | Rationale |
|---|---|
| `ProjectPage` has its own 4-stage layout (not extracted from `NewProjectPage`) | Avoided premature abstraction â€” the two pages share visual structure but have meaningfully different data flows (server hydration vs. blank start). A shared layout component would couple them unnecessarily. |
| Auto-save skips first render via `initialConfigJson` ref comparison | Comparing `JSON.stringify(config)` to the server-returned snapshot is robust against nested object reference changes. The store hydration doesn't write to localStorage (uses `setState` directly), keeping the server as source of truth. |
| `isLoading` initialized to `true` in `userStore` | Prevents premature redirect before auth check completes. Correct behavior for any page that requires authentication â€” show loading state, not login redirect, until the check settles. |
| `page.evaluate` for authenticated project creation in Playwright tests | `page.request` has its own cookie jar separate from the browser context. Only code running inside `page.evaluate` has access to the browser's actual cookie store at the page's origin. |
